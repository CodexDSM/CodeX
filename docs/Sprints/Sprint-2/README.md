---

---

### Objetivo
A segunda sprint focou no desenvolvimento de funcionalidades voltadas à gestão de eventos, automação de cotações e comunicação entre os colaboradores da empresa.

---

### MVP

| Recurso                         | Descrição                                                           |
| ------------------------------- | ------------------------------------------------------------------- |
| Criação de Eventos              | Cadastro de eventos com formulário padrão.                          |
| Automação de Cotação            | Geração automática de cotações para clientes.                       |
| Confirmação/Recusa de Evento    | Colaboradores podem confirmar ou recusar participação em eventos.    |
| Notificações para Colaboradores | Comunicação de eventos e treinamentos ao time.                      |
| Aprovação/Historico de Eventos  | RH acompanham e aprovam eventos realizados.                         |
| Feedback de Evento              | Colaboradores indicam conclusão e escrevem feedbacks.               |
| Agendamento de Follow-up        | Lembretes para vendedores gerenciarem seus clientes.                 |

### Definition of Ready

**Critérios para User Stories**
- Título claro e objetivo compreendido pela equipe
- Critérios de aceitação documentados
- Regras de negócio definidas
- Estimativa feita pelo time
- Sem dependências bloqueadoras
- Equipe compreende totalmente a tarefa

**Critérios para artefatos relacionados**
- Design ou documentação disponível
- Detalhamento das regras de negócio (texto ou diagrama)
- Modelo de dados definido
- Estratégia de testes estabelecida

---

### Definition of Done

- Código testado de acordo com os cenários levantados no backlog
- Upload do código no repositório seguindo padrões de commit da equipe
- Informar o PO sobre a finalização da tarefa para validação
- Registrar o tempo de realização da tarefa no Jira ou diretamente com o PO/SM
