---

---

### Objetivo
A terceira sprint focou no aprimoramento da gestão de fretes, automação de processos operacionais e disponibilização de painéis gerenciais para acompanhamento dos resultados.

---

### MVP

| Recurso                         | Descrição                                                                               |
| --------------------------------| --------------------------------------------------------------------------------------- |
| Exportação de Dados do Dashboard| Permitir gestores exportarem dados do dashboard para Excel ou PDF.                      |
| Dashboard de Fretes Fechados    | Visualizar fretes fechados e receita gerada em painel gerencial.                        |
| Painel Operacional              | Exibir em tempo real os fretes vendidos para acompanhamento operacional.                |
| Agendamento/Follow-up           | Vendedor agenda e recebe lembretes para follow-up com clientes.                         |
| Automação de Cotação            | Automatizar criação de cotações para envio aos clientes.                                |
| Status de Solicitações/Fretes   | Colaborador acessa status das solicitações de frete.                                    |

### Definition of Ready

**Critérios para User Stories**
- Título claro e objetivo compreendido pela equipe
- Critérios de aceitação documentados
- Regras de negócio definidas
- Estimativa feita pelo time
- Sem dependências bloqueadoras
- Equipe compreende totalmente a tarefa

**Critérios para artefatos relacionados**
- Design ou documentação disponível
- Detalhamento das regras de negócio (texto ou diagrama)
- Modelo de dados definido
- Estratégia de testes estabelecida

---

### Definition of Done

- Código testado de acordo com cenários do backlog
- Upload do código no repositório seguindo padrões de commit da equipe
- Informar o PO sobre a finalização para validação
- Registrar o tempo de realização da tarefa no Jira ou com o PO/SM
